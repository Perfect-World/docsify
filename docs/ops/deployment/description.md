### 部署需求说明

#### 1. 服务器环境调研

提前采集服务器基础信息，包括操作系统类型（Windows/Linux）、CPU 配置、磁盘存储空间、外网访问权限、Docker 安装支持性等核心环境指标。

#### 2. 前置准备工作

完成文件目录创建、配置文件与安装包的下载及预配置，确保部署资源就绪。

#### 3. 组件部署流程

按以下顺序执行部署：先安装 Docker 环境，再依次部署 MySQL、Nginx 组件，最后启动 Jar 包应用。

#### 4. 运维脚本配置

需配套完善运维脚本体系，最终实现：

- 拆分独立执行脚本（支持分步骤单独运行）；
- 整合一键式总执行脚本（覆盖全流程自动化部署）；
- 配置 Jar 包专用重启脚本（满足应用运维需求）。
